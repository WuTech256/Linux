#  Compile with gcc and debug with gdb
##  $ gcc -g -o binaryfile source_code.c
    -g tells the compiler to store symbol table in4 in the executable
    this includes:
      + symbol names
      + type info for symbols
      + files and line numbers where the symbols came from
    Tip: Check the size of binary with and without debugging symbols
## $ gdb ./binaryfile 
   or
   $ gdb
   (gdb) file path_to_binary

    gdb is a command line interface.
    This means u will be provided with a prompt at which u can type commands
    The GDB commandline looks like this:
    (gdb)
      
#  Basic Commands
## This starts the program which u want to debug:
  (gdb) run
## To exit gdb, use the quit command (addreviated q) or type an end-of-file character (usually Ctrl-d)
  (gdb) quit
## The "-q" (or "--quiet") option on the command line just tells GDB not to print version information on startup
  (gdb) quiet

# Help
  If u're ever confused about a command or just want more i4, use the "help" command, with or without an argument
  (gdb) help [command]
